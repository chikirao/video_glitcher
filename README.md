# Binary Video Glitch Lab

Локальный вебапп для бинарного глитч-редактирования видео с live preview.

## Что умеет

- Загружает видео и анализирует контейнер перед мутацией байтов.
- Для `MP4` старается трогать только `mdat` payload.
- Для `WebM` работает по payload внутри `Cluster`.
- Для `AVI` смещается в `movi`.
- Показывает превью слева и панель управления справа.
- Пересобирает `Blob` почти в реальном времени при изменении настроек.
- Держит маленький sync-reference оригинала.
- Умеет экспортировать текущую поврежденную версию.
- Пытается ослабить глитч автоматически, если браузер перестал декодировать ролик.

## Как запустить

Лучше запускать через локальный HTTP-сервер, потому что `Web Worker` часто не стартует из `file://`.

Вариант 1:

```powershell
python -m http.server 4173
```

Вариант 2:

```powershell
py -m http.server 4173
```

Потом открыть:

[http://localhost:4173](http://localhost:4173)

## Ограничения

- Это именно бинарный glitch, а не покадровый редактор, поэтому часть файлов все равно будет ломаться на агрессивных настройках.
- Чем выше `Density` и ниже `Guard Rails`, тем выше шанс, что браузер перестанет декодировать ролик.
- Большие видео будут пересобираться медленнее, потому что в памяти клонируется весь файл.
